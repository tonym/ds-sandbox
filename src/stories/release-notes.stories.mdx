<h2>v1.1.0</h2>

<ul>
  <li>GeminiCSS - GeminiCSS is a CSS based style solution derived from Gemini component styles.</li>
  <li>Add icon to Accordion - A start icon and end icon is now available on the Accordion title.</li>
  <li>
    Make Accordion control optional - When the control is hidden, the Accordion is disabled. The use case is for dynamic data in the
    accordion when the data set is empty, and the Accordion should not expand.
  </li>
  <li>
    Replace `async()` in unit tests with `waitForAsync()` - The async method is deprecated, and Angular provides a new, waitForAsync method.
  </li>
  <li>
    Add styles export examples to docs - All Gemini styles are exported, which means you can use Gemini styles without using Gemini
    components. Copy and paste examples are now available in the component API docs.
  </li>
  <li>ButtonGroup component - A styled button group component.</li>
  <li>IconButton component - A dedicated icon button.</li>
  <li>Update core comoponents to use SvgIcon - All Gemini components now use SVG icons instead of icon fonts.</li>
  <li>Contributor documentation - Contributor docs are now available in Storybook.</li>
  <li>Add `gap` property to Flex component - Gap is a fully supported propery of flexbox, and is now available as a property of Flex.</li>
  <li>
    Standardize size selectors across components - All sizes are now standard across components and the theming engine. Legacy sizes are
    still supported, but will be deprecated in Gemini v1.2.0
  </li>
  <li>Add z-index to Sash - Sash now has a z-index for easier use in complex layouts.</li>
  <li>Full color support for Typography - All theme brand color intentions and text color intentions are now supported by Typography.</li>
  <li>Full color support for Button - All theme brand color intentions are now supported by Button.</li>
</ul>
